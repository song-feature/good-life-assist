# 描述

下面希望引入 股价存储 模块

目前的项目中已经有 基于雅虎 api实时获取 股价数据的能力

不过这个实时查询会被限流，数据查询过也没有存储下来，也不利于对于股价进行长久分析

我希望通过 增加一个  股价存储模块 来管理若干支股票的价格记录


# 需求描述

1. 采用数据库记录股价，时间粒度请你自主选择
2. 可以在配置管理中 管理关注的股票列表，添加关注的股票
3. 对于所有关注的股票，应该具备自动抓取最新价格并存储的能力，该功能在系统启动后 就自动开始，抓取频次可以配置
4. 其他模块查询股价时，优先从数据库中查询，如果没有数据，则可以实时查询，但是要具备查询结果可以保存进数据库的能力
5. 该模块是一个子 agent，同时具备下面几个 llm 技能。
   - 列举关注列表 及最新价格(含盘前、前后)
   - 向关注列表添加/删除某支股票
   - 分析某支股票的走势情况
